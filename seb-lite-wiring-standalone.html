No jargon version

Icy Hot is a movement guide: it starts from a known position, watches acceleration, and guides the user with sound and direction.

At the beginning, we tell the system where the user starts and where the target is. After that, the motion sensor feels acceleration, the code estimates where the user has moved, the speaker gives feedback, and the servo acts like a physical compass pointing toward the target.

1. Start and sense

We set the starting point, then read movement

Think of the starting point like setting a map marker. After that, the movement sensor reports acceleration, which is like the body saying, "I just sped up this way."

brain

2. Decide

It estimates position over time

The code keeps updating an estimated path: starting position plus movement over time. It is not magic GPS; it is more like dead reckoning from acceleration.

3. Respond

It gives human feedback

The speaker tells the user how urgent the correction is. The servo acts like an object-oriented compass: it physically points toward the direction the user should care about.

Demo behavior

Movement becomes a path toward the target

The user starts at a known point. Acceleration updates the estimated position. The servo points toward the target. The speaker gives simple feedback: quiet when aligned, slow beep when slightly off, fast beep when drifting, solid tone when the movement needs immediate correction.

1. Starting point

Position is configured at the beginning

The demo starts with a known user position. That gives the system a map origin, like setting the first point before drawing a route.

2. Movement estimate

Acceleration updates the path

The IMU reports acceleration. The firmware uses those readings over time to estimate how the user moved from the start.

3. Human feedback

Sound makes the estimate usable

The speaker uses beep patterns for alignment or correction. Quiet means okay; more urgent sound means the movement/path needs attention.

4. Direction cue

The servo is a physical compass

The servo angle points toward the target/object direction, so the user sees guidance as motion, not as a number.

Rehabilitation layer

Gait and clinical context sit on top of the same movement loop

Once the system can estimate movement and provide feedback, the rehab story becomes clearer: gait patterns, stability, drift, and correction timing can be interpreted clinically. The hardware loop stays simple: movement in, position estimate, speaker feedback, servo direction cue.

How to explain it to a judge

Thirty-second mechanics story

The prototype starts from a configured user position. As the user moves, the motion sensor measures acceleration, and the code estimates position over time. The speaker gives immediate feedback, while the servo acts like a physical compass pointing toward the object or direction of interest. For rehabilitation, the same movement loop can support gait and clinical interpretation.
